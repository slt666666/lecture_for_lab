# 遺伝研スパコンへのログイン

Mac:ターミナル, Windows:PowerShellを開いて下記コマンドを入力します。

```
ssh -i ~/.ssh/id_ed25519_nigsc-gw accountname@gw.ddbj.nig.ac.jp
```

`~/.ssh/id_ed25519_nigsc-gw`の部分はアカウント登録の際に作成したSSH鍵の場所になります。

`accountname`の部分は自分で決めたアカウント名です。

上手くいけば、"Enter passphrase for key ..."と出るので、自分で決めたSSH鍵のパスワードを入力し、Enterを押します。

これで遺伝研のスパコンへ接続できます。

次に、プログラムや計算等を実行する所へ接続します。

```
ssh a001
```

というコマンドを実行してください。

`accountname@a001:~$`と表示されていれば成功です。この状態で様々な解析などを進めていく事になります。

# Linuxの基本コマンド

遺伝研のスパコンではLinuxというOSが動いており、コマンドで操作をすることになります。

そこでまず、Linuxの基本的なコマンドをいくつか紹介します。

基本的なコマンドを使用していく中で、コマンドを使用する感覚がつかめるかと思います。

では代表的なコマンドいくつか見ていきましょう。

## pwd 現在のディレクトリの表示

`pwd`コマンドで自分が今いる位置が分かります。

(Print Working Directoryの略です。)

## ls ファイル・ディレクトリの一覧の表示

(Listの略)ファイル等の一覧を表示することができます。

コマンドにはオプションというものを設定することができ、いろいろ機能を追加したり変更できます。

lsコマンドにもオプションがあり、-aとつけると隠しファイルなども含めて表示してくれます。

-lというオプションをつけるとより細かい情報を表示してくれます。

```
ls -a
ls -l
ls -la
```

等、色々試してみてください。

バイオインフォマティクスのツールを使用する際にもコマンドに色々オプションを設定することになります。

## touch ファイルがあれば日時を更新、無ければ空のファイルを作る

`touch`コマンドで中身が空のファイルを作成することができます。

```
touch <好きなファイル名>
```
(好きなファイル名の部分には"好きな文字列.txt"と入力してみてください。)

新しく自分のつけた名前のファイルが作成されたかと思います。`ls`コマンドで見てみましょう。

今回のファイル名の様にコマンドの後にスペースで区切って入力するものを引数と呼びます。

今回の様な任意の文字列を指定する場合や、コマンドを適用するファイルを指定する場合に引数の形で使用します。

※作成したファイルの編集については、vimコマンド等を使用するとコマンドから中身を編集できますが、ちょっと難易度が高いので今回は割愛します。

## mkdir ディレクトリの作成

`mkdir`コマンドで新しいディレクトリ(フォルダ)を作成することができます。

こちらも先ほどのtouchコマンド同様、引数にファイル名を指定します。

```
mkdir <好きなフォルダ名>
```

(好きなフォルダ名の部分は好きな文字列を入力してください。)

作成したら、また`ls`コマンドでフォルダが作成されたか確認してみてください。

## cd 現在のディレクトリの変更

`cd`コマンドで今いる場所を移動できます。

(chage directoryの略です。)

```
cd <行き先の名前>
```

という形で使用します。

例えば、さきほど作成したフォルダの中に移動してみましょう。

```
cd sample
```
(`sample`の部分には自分の作成したフォルダ名を入れる)

と入力してEnterを押すとsampleというフォルダに移動できます。

この状態でlsをすると、sampleフォルダ内のファイルが表示されるかと思います。

```
cd ..
```

この様な`..`へ移動する様なコマンドは1つ前のフォルダに戻ることを意味します。

また`cd`のみを実行すると、起動した時のスタート位置に戻ります。

## mv ファイルやディレクトリの移動

`mv`コマンドでファイルやフォルダを任意の場所に移動させることができます。

先ほどの`touch`コマンドで作成しファイルを作成したフォルダ内に移動させてみましょう。

```
mv <ファイル名> <フォルダ名>
```

上のコマンドを動かすと先ほど作成したファイルがフォルダ内に移動したかと思います。

また、ファイル名を変える際にも`mv`コマンドが使用できます。

```
mv <ファイル名> <新しいファイル名>
```

## rm ファイルを削除する

`rm`コマンドでファイルやディレクトリを削除することができます。

こちらも引数に削除したいファイル名を指定します。

```
rm <削除したいファイル名>
rm -r <削除したいディレクトリ名>
```

※ディレクトリ(フォルダ)を削除する場合はオプションで-rが必要です。

## wget インターネットからファイルをダウンロードする

`wget`コマンドを使用することで、URLを通じてインターネット上のファイルをダウンロードできます。

例えばネット上に下記の様なデータがあります。

データのURL: https://raw.githubusercontent.com/CropEvol/lecture/a996a8d65eabee7b1dd1204b77f0ea90bc2dba68/data/membrane_dataset.csv

このデータをコマンドでダウンロードする場合、`wget`コマンドの引数にURLを指定すればOKです。

```
wget <URL>
```

今回の場合、

```
wget https://raw.githubusercontent.com/CropEvol/lecture/a996a8d65eabee7b1dd1204b77f0ea90bc2dba68/data/membrane_dataset.csv
```

となります。

`ls`をしてみると`membrane_dataset.csv`というファイルが今いる場所にダウンロード出来ているかと思います。

## less テキストファイルを表示・閲覧する

`less`コマンドでファイルの中身を見ることができます。

例えば先ほどダウンロードしたファイルの中身を見るには、

```
less membrane_dataset.csv
```

とすれば中身を見ることができます。

元の画面に戻るにはqを押すと戻ります。

`less`コマンドにもオプションがあり、`-N`とつけることで行番号を表示したり、`-S`とつけることで改行無しで表示させられたりします。

```
less -S membrane_dataset.csv
```

## apt Linuxのソフトウェア管理コマンド(※遺伝研スパコンでは使用できない)

`apt`コマンドを使用することで、Linuxで使用できるソフトをインストールしたり更新したりすることができます。

例えば`screenfetch`というコマンドがLinuxにはありますが、

今`screenfetch`と打ってもcommand not found(そんなコマンドは無い)と出てしまうと思います。

そこでaptコマンドでこのツールをインストールしてみます。

```
apt install screenfetch
```

これで`screenfetch`コマンドが使えるようになったかと思います。

※`screenfetch`コマンドはOSやPCスペックの情報などを表示してくれます

`apt install`コマンドを使用して、もともと入っていないコマンドやバイオインフォマティクス用のツールも一部インストールすることができます。

## clear 画面をきれいにする

`clear`コマンドを使用すると画面に表示されていたものを消して綺麗にすることができます。

(表示が消えるだけで実行した内容はそのまま)

他にも様々なコマンドがありますが、あくまでPCを操作するためのコマンドになるので、このあたりにしておきます。

下記サイトの様なコマンドを紹介してくれているサイトもたくさんあるので、気になる人はまた見てみてください。

[初心者向け！基本的なLinuxコマンドと使い方！](https://qiita.com/nokonoko_1203/items/35c4078bf72cffb16171)

## シェルスクリプトの使い方

先ほどまではコマンドを1つずつ(1行ずつ)実行していましたが、複数のコマンドを一気に実行することも可能です。

この時にシェルスクリプトというものを使用します。

実行するコマンドを全て1つのファイルに記入し、実行する形になります

ひとまず簡単な例を実行してみます。

先ほどの`wget`コマンドを使用して、

Shell スクリプト: https://github.com/slt666666/lecture_for_lab/blob/main/data/shell_test.sh

をダウンロードしてください。

ダウンロードしたtest.shには下の様な内容が書いてあります。

```
mkdir shell_test
cd shell_test
touch text1.txt
touch text2.txt
touch text3.txt
cd ..
```

中身はshell_testというフォルダを作成し、その中に移動、そこでtext1~3という3つのファイルを作成する。というコマンドが書かれています。

このtest.shを実行してみましょう。実行するには`bash`コマンドを使用します。

```
bash test.sh
```

という形で実行することができます。

この様な形で、ファイルに複数のコマンドを書いておけば、1度に実行することができます。

大量のデータをダウンロードしたり、大量の配列データを処理する場合に役立ちます。

# スパコンの使い方

## スパコンにJobを入れる

コマンドを使用して、スパコンを操作できるようになりましたが、

この状態で長時間かかる計算や解析を実施するにはスパコンへ接続し続ける必要が出てきます。

スパコンにはJobシステムというものがあり、作業をスパコンに投げてスパコンが空き次第、裏で実行させておく事が出来ます。

さっき扱ったShellスクリプトの様なスクリプトを書いて実行する形になります。

下記コマンドで一例`job_script.sh`をダウンロードしてください。

Jobスクリプト: https://raw.githubusercontent.com/slt666666/lecture_for_lab/refs/heads/main/data/job_script.sh

`less`コマンドで中身を見てみると以下の様な内容が書かれています。

```
#!/bin/bash
#SBATCH -p epyc                     -> どのサーバーを使用するか
#SBATCH -t 0-00:10:00               -> 確保時間
#SBATCH --mem-per-cpu 4g            -> CPUあたりのメモリ(これは4GB)
#SBATCH -J an_example               -> ジョブ名
 
...略
```

CPUあたりのメモリや確保時間を大きくすると、中々ジョブが動かなくなります。

実際にこのスクリプトをJobとして実行するには`sbatch`コマンドで動かします。

```
sbatch job_script.sh
```

上手くJobが入っていれば、`job_test`というフォルダを作成し、その中に10秒に1ファイルを作成する、という処理が動いているはずです。

また、今自分がどんなJobを入れているかは

```
squeue -u accountname
```

で確認する事が出来るので、うまくJobが入っていれば何か表示されているはずです。

Jobスクリプトは、複数のCPUを使用したり、他にも色々細かく設定できますが、詳しくは　https://sc.ddbj.nig.ac.jp/guides/software/JobScheduler/Slurm/　のバッチジョブやパラレルジョブの項を参照してください。


## 自分のPCから遺伝研スパコンへのデータ移行

自分のPCと遺伝研スパコンとのデータのやり取りには`FileZilla`というプログラムを使用すると便利です。

https://filezilla-project.org/

ダウンロード・インストールしてください。直感的にデータのやり取りが出来るようになります。

接続方法は

1. 左上のSite Managerをクリック
2. New Siteから新しい接続先を作成、設定は↓の通り
* Protocol: SFTP
* Host: gw.ddbj.nig.ac.jp
* Logon Type: Key file
* User: アカウント名
* Key file: ssh鍵のファイル(.pubが無い方)を選択すると、`.ppk`をファイルを作成する様な指示が出るので、指示に従って作成し、それを選択。

3. 作成出来たら右下のConnectボタンで接続できます。

## 練習:シーケンスデータ(fastqファイル)のクオリティコントロールの実行

次世代シーケンスを実施するとfastqファイルが得られます。

このファイルはそのまま使用するのではなく、通常クオリティの低いリードを省いたりするのクオリティコントロールを実行します。

以下のデータをスパコン上に入れて、

URL: https://github.com/slt666666/lecture_for_lab/raw/refs/heads/main/data/sample.fastq.gz

クオリティコントロール用の`fastp`というプログラムが遺伝研スパコンではもともと使用出来ると思うので、

下記コマンドをJobとして実行してください。

```
fastp -i sample.fastq.gz -o sample_filtered.fastq.gz -w 3 -q 15 -n 10
```

上手く行けば`sample_filtered.fastq.gz`というファイルが出来ているはずです。

手順

1. `wget`コマンドで`sample.fastq.gz`をダウンロード。
2. さきほどの`job_script.sh`をFileZillaで自分のPCに持ってくる。
3. 自分のPCで`job_script.sh`を編集、"ここより後に実行したいプログラムを書く"以降を削除し、代わりに上のコマンドを書いて保存。
4. FileZillaで`job_script.sh`を遺伝研スパコンに再び戻す。
5. `sbatch`コマンドで実行する。

ただ、わざわざ自分のPCを経由させなくても、スパコン内のコマンドでファイルを直接編集することも可能です。

'vim'コマンド等で実施できます。使い方は自分で調べてみてください。

## プログラムのインストールについて

バイオインフォ系の解析をする際に、新しくツールをインストールする必要がある場合があります。

遺伝研のスパコンには元々ある程度基本的なプログラムは揃っていて、最初からコマンドとして使えるツールが多い気がしますが、

もし新しくプログラムをインストールする必要が出てきた場合には、基本的にプログラムのWebページに

`Installation`みたいなところがあるので、それに従ってインストールをします。
